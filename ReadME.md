# Real-Time Web Anomaly Detection

This project provides a full-stack, real-time Machine Learning pipeline for detecting volumetric web attacks (like DDoS, brute force, and scraping) by analyzing Nginx/Apache log files in real-time.

## Architecture
1. **Ingestor (`ingestor.py`)**: Uses the `watchdog` library on a daemon thread to tail `log.file` continuously.
2. **Preprocessor (`preprocessor.py`)**: Parses standard web log lines into Pandas DataFrames.
3. **Feature Extractor (`features.py`)**: Aggregates raw logs into 5-minute rolling windows, calculating critical behavioral metrics like `err_4xx_pct`, `err_5xx_pct`, and `unique_ips`.
4. **Detector (`detector.py`)**: Implements an unsupervised ML ensemble using a hard-majority vote between:
   - `IsolationForest`
   - `OneClassSVM`
   - `LocalOutlierFactor`
   *(Data is scaled using `RobustScaler` to remain immune to the massive outliers generated by DDoS spikes).*
5. **Dashboard (`dashboard.py`)**: A Streamlit application that natively wires into the live ingestion and ML pipeline using `st.session_state`.
6. **Data Store (`store.py`)**: Automatically pushes any detected anomalies directly to a Supabase PostgreSQL database.
7. **REST API (`api.py`)**: A FastAPI service that exposes the Supabase data to downstream consumers.

## Setup
Make sure you have Poetry or `pip` configured with the requirements. 
Dependencies: `pandas`, `scikit-learn`, `watchdog`, `streamlit`, `fastapi`, `uvicorn`, `supabase`, `python-dotenv`.

Ensure you have a `.env` file in the root with:
```env
NEXT_PUBLIC_SUPABASE_URL=<your_url>
SUPABASE_SERVICE_ROLE_KEY=<your_key>
```

## Running the Project

**1. Run the Streamlit Dashboard (Live ML Pipeline)**
```bash
streamlit run dashboard.py
```
*(Append new lines to `log.file` to see them processed in real-time on the UI)*

**2. Run the FastAPI Server (Anomaly REST API)**
```bash
uvicorn api:app --reload
```

**3. Run the End-to-End Integration Test**
```bash
python test_pipeline.py
```
